# Что делать дальше — пошагово

Ты написал код. Теперь нужно: положить его на GitHub, поднять сервер на Railway и подключить бота. Ниже — что делать по шагам.

---

## Шаг 1. Залить код на GitHub

1. Зайди на [github.com](https://github.com) и войди в свой аккаунт.
2. Нажми зелёную кнопку **«New»** (новый репозиторий).
3. Назови репозиторий, например: `collective-app`. Нажми **Create repository**.
4. Открой папку своего проекта на компьютере в терминале (PowerShell или командная строка).
5. Выполни по очереди (подставь свой логин и название репо):

```bash
git init
git add .
git commit -m "Первый коммит"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/collective-app.git
git push -u origin main
```

Готово: код теперь на GitHub.

---

## Шаг 2. Поднять проект на Railway

1. Зайди на [railway.app](https://railway.app) и войди (можно через GitHub).
2. Нажми **«New Project»**.
3. Выбери **«Deploy from GitHub repo»** и укажи свой репозиторий `collective-app`.
4. Railway сам увидит `Dockerfile` и начнёт сборку. Дождись, пока деплой закончится (зелёная галочка).
5. Railway даст тебе ссылку на приложение, например: `https://collective-app-xxxx.up.railway.app`. Запомни её — это твой **домен**.

---

## Шаг 3. Настроить базу данных (PostgreSQL)

1. В том же проекте на Railway нажми **«New»** → **«Database»** → **PostgreSQL**.
2. Подожди, пока база создастся.
3. Зайди в созданную базу, открой вкладку **«Variables»** или **«Connect»** и скопируй строку подключения (URL). Она выглядит примерно так:  
   `postgresql://user:password@host:port/railway`
4. Вернись в своё приложение (первый сервис, не база). Открой его → вкладка **«Variables»**.
5. Добавь переменные (подставь свои значения):

| Имя | Значение |
|-----|----------|
| `DATABASE_URL` | `postgresql+asyncpg://user:password@host:port/railway` (в начале добавь **+asyncpg** после postgresql) |
| `SCHEDULER_JOBSTORE_URL` | `postgresql+psycopg2://user:password@host:port/railway` (тот же хост, но **+psycopg2**) |
| `TELEGRAM_BOT_TOKEN` | токен от @BotFather |
| `TELEGRAM_WEBHOOK_PATH` | например: `/telegram/webhook/мой_секретный_ключ` |
| `WEBHOOK_BASE_URL` | твой домен Railway, например: `https://collective-app-xxxx.up.railway.app` |
| `JWT_SECRET` | любая длинная случайная строка (придумай или сгенерируй) |
| `ADMIN_USERNAME` | логин для админки, например: `admin` |
| `ADMIN_PASSWORD_HASH` | хеш пароля (см. ниже) |

**Как сделать хеш пароля:** на своём компьютере в терминале выполни (подставь свой пароль):

```bash
python -c "import passlib.hash; print(passlib.hash.bcrypt.hash('твой_пароль'))"
```

Скопируй результат в `ADMIN_PASSWORD_HASH`.

6. Сохрани переменные. Railway перезапустит приложение с новыми настройками.

---

## Шаг 4. Запустить миграции (создать таблицы в БД)

1. В Railway открой свой сервис приложения (не базу).
2. Найди раздел **«Settings»** или **«Deploy»** — там обычно есть возможность выполнить команду (Console / Shell / one-off command). Или используй **«Deploy» → «Custom start command»** только для одного запуска миграций.
3. Нужно один раз выполнить команду миграций. Варианты:
   - Если в Railway есть **Shell/Console** в сервисе — открой его и выполни:
     ```bash
     alembic upgrade head
     ```
   - Либо добавь в **Start Command** перед uvicorn что-то вроде:  
     `alembic upgrade head && uvicorn ...`  
     (точная команда зависит от того, как у тебя настроен старт в Railway.)
4. После успешного выполнения таблицы в базе созданы.

---

## Шаг 5. Подключить бота к Telegram

1. Открой своего бота в Telegram (найди по имени).
2. Напиши ему **/start**. Сервер уже должен быть запущен на Railway, и при первом запросе приложение само вызовет Telegram API и «привяжет» webhook к твоему домену (потому что ты указал `WEBHOOK_BASE_URL`).
3. Если бот не отвечает:
   - Проверь, что в Variables указан правильный `TELEGRAM_BOT_TOKEN` и `WEBHOOK_BASE_URL` без лишнего слэша в конце.
   - Посмотри логи сервиса в Railway — там будет видно, успешно ли установился webhook.

## Шаг 6. Открыть Mini App и админку

1. **Mini App для пользователей:**  
   В настройках бота в BotFather можно добавить кнопку меню или команду, которая открывает ссылку:  
   `https://твой-домен.up.railway.app/mini-app/`  
   Тогда пользователи смогут заходить в каталог и заказы прямо из Telegram.

2. **Админка для тебя:**  
   Открой в браузере:  
   `https://твой-домен.up.railway.app/admin-ui/`  
   Войди логином и паролем, которые зашиты в `ADMIN_USERNAME` и в хеше `ADMIN_PASSWORD_HASH`. Там можно смотреть заказы и менять их статусы.

## Фото товаров: почему не отображаются и что сделать

**Почему так бывает:** загруженные через админку фото сохраняются в папку `uploads/` на сервере. На Railway контейнер при каждом перезапуске или новом деплое создаётся заново, и эта папка очищается — старые файлы пропадают, ссылки вида `/uploads/xxx.jpg` начинают отдавать 404, и в приложении вместо фото показывается плейсхолдер 🪑.

**Что можно сделать:**

1. **Включить постоянный том (Volume) на Railway**  
   Тогда папка с загрузками не будет очищаться при перезапуске:
   - В проекте Railway открой свой сервис (приложение).
   - Зайди в **Settings** → раздел **Volumes**.
   - Нажми **Add Volume**, укажи путь монтирования: **`/app/uploads`** (именно этот путь использует приложение).
   - Сохрани и при необходимости перезапусти сервис.  
   После этого всё, что загружается через «Загрузить фото» в админке, будет храниться в этом томе и фото перестанут пропадать.

2. **Использовать внешние ссылки на фото**  
   Вместо загрузки файла в админке можно вставить **прямую ссылку** на картинку (например с Imgur или Telegra.ph) в поле «Фото товара». Такие ссылки не зависят от сервера и будут работать и после перезапусков.

3. **В приложении**  
   Если картинка по какой-то причине не загрузилась (404, ошибка сети), вместо «битой» иконки теперь показывается плейсхолдер 🪑 — интерфейс остаётся аккуратным.

## Если что-то пошло не так

- **Бот не отвечает** — смотри логи в Railway и проверь `TELEGRAM_BOT_TOKEN` и `WEBHOOK_BASE_URL`.
- **Ошибка при открытии сайта** — проверь, что деплой завершился и в Variables нет опечаток.
- **«Internal error» или ошибки БД** — убедись, что миграции выполнены (`alembic upgrade head`) и оба URL (`DATABASE_URL` и `SCHEDULER_JOBSTORE_URL`) ведут на одну и ту же базу Railway.

Дальнейшие действия — это уже использование: добавлять товары (через админку или API), тестировать заказы через Mini App и при необходимости смотреть логи и переменные в Railway.
