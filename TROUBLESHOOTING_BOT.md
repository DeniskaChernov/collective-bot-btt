# Бот не отвечает — что проверить

Если ты отправил боту `/start`, а он молчит, пройди по пунктам ниже.

---

## 1. Приложение на Railway вообще запущено?

- Зайди в [railway.app](https://railway.app) → свой проект → сервис с приложением (не базу).
- Статус должен быть зелёный (Running). Если «Crashed» или постоянно перезапускается — смотри **логи** (вкладка Deploy → View Logs).
- Открой в браузере: `https://ТВОЙ-ДОМЕН.up.railway.app/health`  
  Должен вернуться JSON вроде `{"ok":true,"result":true}`. Если страница не открывается — приложение не работает или домен неверный.

---

## 2. В переменных задан WEBHOOK_BASE_URL

Без этого Telegram не знает, куда слать сообщения — webhook не устанавливается.

- В Railway: сервис приложения → **Variables**.
- Должна быть переменная **`WEBHOOK_BASE_URL`** со значением твоего домена **без слэша в конце**, например:
  ```text
  https://collective-app-production-xxxx.up.railway.app
  ```
- Если этой переменной нет или она пустая — добавь/исправь, сохрани. Railway перезапустит приложение.

---

## 3. TELEGRAM_WEBHOOK_PATH указан верно

- В Variables должна быть **`TELEGRAM_WEBHOOK_PATH`** — путь, по которому Telegram шлёт обновления.
- Значение должно **начинаться с `/`**, например:
  ```text
  /telegram/webhook/мой_секретный_ключ
  ```
- После изменения переменных подожди перезапуска и снова напиши боту `/start`.

---

## 4. Токен бота правильный

- Переменная **`TELEGRAM_BOT_TOKEN`** должна быть тем токеном, который дал @BotFather именно для бота «Bententrade | Collective».
- Без пробелов и кавычек в начале/конце. После правки — сохранить переменные и дождаться перезапуска.

---

## 5. Логи приложения

В Railway: сервис приложения → **Deploy** → **View Logs**.

- При успешном запуске и установке webhook должно быть что-то вроде:
  ```text
  telegram.webhook_set {"url": "https://твой-домен/telegram/webhook/..."}
  ```
- Если видишь:
  ```text
  telegram.webhook_skipped ... WEBHOOK_BASE_URL is not set
  ```
  значит webhook не ставится — добавь/проверь **WEBHOOK_BASE_URL** (п. 2).
- Если есть ошибки (красные строки, traceback) — приложение падает до обработки сообщений. Часто это неверный `DATABASE_URL`, отсутствие миграций или другая ошибка при старте.

---

## 6. Secret Token (если используешь)

Если в Variables задан **`TELEGRAM_SECRET_TOKEN`**, тот же токен нужно указать в BotFather:

- Открой @BotFather → выбери бота → **Bot Settings** → **Secret Token** → вставь то же значение.

Иначе Telegram будет слать запросы без нужного заголовка, и приложение будет отвечать 403 — обновления до бота не дойдут.

---

## Краткий чек-лист

| Проверка | Где смотреть |
|----------|----------------|
| Сервис в статусе Running | Railway → сервис приложения |
| Открывается `/health` | Браузер: `https://твой-домен/health` |
| Задан `WEBHOOK_BASE_URL` (без слэша в конце) | Railway → Variables |
| Задан `TELEGRAM_WEBHOOK_PATH` (начинается с `/`) | Railway → Variables |
| Верный `TELEGRAM_BOT_TOKEN` | Railway → Variables |
| В логах есть `telegram.webhook_set` | Railway → View Logs |
| Если есть `TELEGRAM_SECRET_TOKEN` — он совпадает с BotFather | BotFather → Bot Settings → Secret Token |

Чаще всего бот молчит из‑за **не заданного или неверного `WEBHOOK_BASE_URL`** — тогда webhook не устанавливается и Telegram никуда не отправляет обновления. Исправь переменные, дождись перезапуска и проверь логи и `/health` ещё раз.
